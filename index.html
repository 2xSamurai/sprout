Collection of React Native Components

No, this is not an npm package and wont be made into one anytime soon.

Most of the time when using an npm package I found myself only using a small part of it and not the whole thing. So why not make a collection where you can just copy paste the things you want and configure it to your needs. This will keep your code clean.

Also you wont have to worry about useless stuff while updating your codebase.

PS: This is not for the seasoned developer. But for the ones starting out.

Onboarding Component

Highlights of the component

  1. Custom Slider using FlatList
  2. Skip Button
  3. Next Button
  4. Custom Components for each slide
  5. Redux State
Component names in quotes are the actual names used in the project for ease of locating them
TIP: Press cmd + p(on Mac) or cntrl + p(on Windows) to bring up quick search option if you are using VSCode. Input the filename and locate them easily.

Custom Slider using FlatList

Onboarding screen uses a "Slider.js" component for the sliding functinality. Under the hood the the "Slider.js" component uses a simple FlatList. If needed you can further extend the functinalities of FlatList.

Skip Button

Simple set the status of onboarding state in redux to false. Thereby hiding the onboarding screens.

Learn to set up a basic Redux implementation here

Next Button

Uses the Flatlist's built-in scrollToIndex functionlity to slide to the next slide. Inside "Slider.js" is a scrollTo function which handles the logic.

Custom Components for each slide

Open "index.js" inside the onboarding folder. The number of slides to show is defined in this file. 

const onboardingItems = [
        { id: 1, component: <StepOne /> },
        { id: 2, component: <StepTwo />  },
        { id: 3, component: <StepThree />  },
    ]

StepOne, StepTwo and StepThree are different componets used as slides for onboarding. In this template the header and footer are static though.
